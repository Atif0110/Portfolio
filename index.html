AI Customer Churn Intelligence Platform

Demo GitHub

An end-to-end Decision Intelligence platform that goes beyond churn scores — it combines Logistic Regression with a Groq-powered Llama 3.1 LLM to predict customer risk, explain the drivers behind it, and generate targeted retention strategies. Built with a modular FastAPI backend, provider-agnostic LLM abstraction layer, and an interactive Streamlit analyst dashboard.

95% Accuracy FastAPI Backend Batch Inference Llama 3.1 via Groq Scenario Simulation
Python 3.10+ FastAPI scikit-learn Streamlit Groq API Llama 3.1 Matplotlib python-dotenv Uvicorn
Architecture & Technical Highlights
  • Modular FastAPI backend with four services: DS service, GenAI service, Orchestrator, and History logging — each with clear separation of concerns
  • Logistic Regression model predicts churn probability using usage hours, support ticket count, and tenure; outputs risk tiers (High >60%, Medium 30–60%, Low <30%)
  • Rule-based driver detection identifies interpretable churn signals such as low product usage, high support volume, and short customer tenure
  • Provider-agnostic LLM factory layer supports Groq, OpenAI, and Gemini — swappable without touching application logic
  • /simulate endpoint models intervention impact, e.g. reducing a 78% risk score to 51% by simulating proposed retention actions
  • Batch CSV processing pipeline for bulk scoring with explainability output per customer row
  • Low-temperature LLM prompting ensures consistent, structured retention strategy generation suitable for analyst consumption
FastAPI Machine Learning Explainability LLM Integration Groq Streamlit
Forecasting · Quantitative Analytics · RAG

Energy Market Forecasting & LLM Insights System

GitHub

A complete end-to-end electricity price forecasting and algorithmic trading pipeline. Ingests day-ahead market data via ENTSO-E API (with a synthetic fallback for reproducibility), runs thorough QA checks, trains an XGBoost model with rich temporal features, translates forecasts into LONG/SHORT/NO-TRADE signals with confidence intervals, simulates PnL, and generates AI-assisted trader reports via Groq.

20–35% Better Forecasting ENTSO-E API Integration PnL Simulation Confidence Intervals AI Trader Reports
Python 3.10+ XGBoost ENTSO-E API Groq API pandas scikit-learn Plotly (HTML output) YAML config
Architecture & Technical Highlights
  • Five-module pipeline: data_ingestion.py → qa_pipeline.py → forecasting.py → trading_view.py → llm_analyst.py, fully config-driven via config.yaml
  • Data QA framework performs time-integrity checks (missing/duplicate timestamps), statistical outlier detection, and economic sanity checks (price vs. demand positive relationship, price vs. renewables negative relationship)
  • XGBoost model trained on engineered features: hour-of-day, weekday, lag-1h, lag-24h, lag-168h, rolling mean/std, and momentum indicators against a lag-24 baseline
  • Trading signal logic: LONG when forecast > reference + uncertainty band, SHORT when forecast < reference − uncertainty, NO TRADE within band
  • Interactive HTML output (outputs/charts/trading_view.html) with forecasts, signals, and cumulative PnL; submission.csv exports predictions with confidence intervals
  • AI analyst module (Groq-powered) is optional and automatically skipped if no API key is provided, ensuring full reproducibility with synthetic data

Urban Analytics Real-Time Intelligence Dashboard

Demo

A real-time urban intelligence platform that fuses live traffic, weather, and environmental data from multiple public APIs into a single monitoring and forecasting layer. Features automated schema validation, anomaly detection, statistical forecasting, and interactive multi-city geospatial visualizations — all refreshing on a 3-minute pipeline cycle.

3-Min Refresh Pipeline Multi-Source API Fusion Geospatial Analytics Anomaly Detection Live Monitoring
Python Streamlit REST APIs pandas Geospatial Libs Statistical Forecasting Caching Layer
Architecture & Technical Highlights
  • Multi-source real-time data ingestion from traffic, weather, and environmental APIs with automated schema validation on each pull cycle
  • Caching layer prevents redundant API calls between 3-minute refresh windows, reducing latency and rate-limit exposure
  • Anomaly detection flags statistically unusual readings across all data streams and surfaces them in the live monitoring view
  • Statistical forecasting module projects near-term trends for key urban metrics — used for predictive alerts in the dashboard
  • Interactive multi-city geospatial visualizations allow analysts to switch between cities and drill into district-level data
  • Live monitoring workflow refreshes the full pipeline on schedule, aggregates new data, re-runs anomaly checks, and updates all dashboard panels
