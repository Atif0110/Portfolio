European Power Market Forecasting System
Code

End-to-end pipeline for day-ahead electricity price forecasting using real ENTSO‑E market data. Engineers 12+ features (lag‑1/24/168h, rolling volatility, calendar effects) and trains an XGBoost model achieving 20–35% improvement over baseline, outperforming 3 alternative models evaluated during development. Includes synthetic fallback data for reproducibility.

20–35% Improvement 12+ Features 3 Models Outperformed PnL Simulation 12+ Months Backtest Zero QA Incidents
Python XGBoost scikit-learn Pandas NumPy Plotly ENTSO‑E API Groq API Prompt Engineering YAML Config
📐 Architecture & Engineering Decisions
  • Five-module pipeline: data_ingestion.py → qa_pipeline.py → forecasting.py → trading_view.py → llm_analyst.py.
  • 8-point automated QA: Time‑integrity checks, statistical outlier detection, economic sanity (demand‑price relationship), and freshness monitoring — reduced data quality incidents to zero.
  • Feature engineering: 12+ features including lag‑1/24/168h, rolling volatility, calendar effects, and momentum.
  • Trading logic: LONG if forecast > reference + confidence band, SHORT if < reference – band, HOLD otherwise.
  • LLM agent: Programmatically integrated via API with structured prompt engineering to auto-generate trader briefs — eliminating 3+ hours of manual report writing per week across 5 recurring report types.
  • Backtesting: PnL simulation across 12+ months of historical market data.
  • Reproducibility: Synthetic fallback dataset if ENTSO‑E API key is not provided.

95% Accuracy 100+ Customers <5ms Inference FastAPI + Uvicorn What‑If Simulation Provider‑Agnostic LLM

  • Four-service modular design: DS Service (inference), GenAI Service (LLM), Orchestrator, History Logger — clean separation of concerns.
  • Logistic Regression: Trained on usage hours, support tickets, and tenure; outputs probability + risk tier (High >60%, Medium 30–60%, Low <30%).
  • Provider-agnostic LLM factory: Swappable between Groq, OpenAI, Gemini without touching business logic — currently using Groq Llama 3.1 8B.

  • What‑If simulation: /simulate endpoint models intervention impact for self-service scenario analysis — reducing ad-hoc analyst requests.
  • Batch processing: CSV ingestion with per‑customer predictions and explainability scores.

Real‑Time Urban Analytics Dashboard
Live App

Architected an automated multi-source data pipeline integrating 3 live REST APIs (OpenWeatherMap, TomTom Traffic, WAQI Air Quality) with schema validation, error handling, and 3‑minute refresh caching — maintaining 99% data availability across continuous downstream modelling. Features interactive pydeck maps with TomTom traffic flow overlay, ARIMA-style 30-minute traffic forecasting, anomaly detection, and automatic alerts for heat, congestion, and pollution.

3 Live APIs 3‑Min Refresh 99% Availability ARIMA Forecasting Anomaly Detection 5+ Metrics Auto Alerts
Python Streamlit Pandas NumPy REST APIs pydeck OpenWeatherMap TomTom API WAQI API Statistical Forecasting Caching Plotly
📐 Architecture & Engineering Decisions
  • Multi-source ingestion: Automated schema validation on each refresh cycle using shared requests.Session for efficiency.
  • Caching layer: Streamlit's @st.cache_data(ttl=180) ensures 3‑minute API refresh — prevents rate-limit exposure.
  • 30-minute traffic forecast: Trend extrapolation (ARIMA-style) for near-term predictive alerts.
  • Anomaly detection: Flags statistically unusual readings and surfaces in live monitoring.
  • Visualisation: Interactive pydeck maps with OpenStreetMap + TomTom live traffic flow overlay — multi-city comparison support.
  • Alert system: Auto-triggered warnings for temperature >40°C, PM2.5 >100 µg/m³, and speed <20 km/h.
