Backend · Decision Intelligence · LLM Integration
AI Customer Churn Intelligence Platform
Demo Code

Engineered a modular FastAPI backend that combines a logistic‑regression churn predictor with a Groq‑powered Llama 3.1 LLM layer. The system exposes REST endpoints for prediction, driver explanation, and scenario simulation. Built with a provider‑agnostic LLM factory, batch CSV processing, and a Streamlit consumption dashboard.

92% Accuracy FastAPI + Uvicorn Batch Inference Groq API Scenario Simulation
Python FastAPI Uvicorn scikit-learn Streamlit Groq API Llama 3.1 Pandas Matplotlib
📐 Architecture & Engineering Decisions
  • Four‑service modular design: DS Service (inference), GenAI Service (LLM), Orchestrator, History Logger — clear separation of concerns.
  • Logistic Regression trained on usage hours, support tickets, tenure; outputs probability + risk tier (High >60%, Medium 30–60%, Low <30%).
  • Driver detection identifies interpretable signals (low usage, high support, short tenure) for actionable insights.
  • Provider‑agnostic LLM factory supports Groq, OpenAI, Gemini — swappable without touching business logic.
  • /simulate endpoint models intervention impact (e.g., 78% → 51% risk by simulating actions).
  • Batch CSV processing with per‑customer explainability scores.
  • Streamlit dashboard for analyst consumption, deployed on Streamlit Cloud.
Time‑Series Pipeline · Quantitative Engineering
European Electricity Price Forecasting & Trading Pipeline
Code

Built an end‑to‑end data pipeline for day‑ahead electricity price forecasting using XGBoost. Ingests ENTSO‑E API data, runs rigorous QA checks, engineers temporal features, generates LONG/SHORT/HOLD signals with confidence intervals, simulates trading PnL, and produces AI‑assisted trader reports.

20–35% Improvement PnL Simulation Confidence Intervals Trading Signals
Python XGBoost scikit-learn Pandas NumPy Plotly ENTSO‑E API Groq API YAML Config
📐 Architecture & Engineering Decisions
  • Five‑module pipeline: data_ingestion.py → qa_pipeline.py → forecasting.py → trading_view.py → llm_analyst.py.
  • QA framework: time‑integrity checks, statistical outlier detection, economic sanity (demand‑price relationship).
  • Feature engineering: hour‑of‑day, weekday, lag‑1h/24h/168h, rolling mean/std, momentum vs lag‑24 baseline.
  • Trading logic: LONG if forecast > reference + band, SHORT if < reference – band, HOLD otherwise.
  • Interactive Plotly HTML output (outputs/charts/) with cumulative PnL visualisation.
  • CSV submission with predictions + confidence intervals.
  • Optional Groq analyst (auto‑skipped if no API key) for reproducibility.
